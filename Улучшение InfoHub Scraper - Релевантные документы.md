# Цель
Настроить scraper для сбора релевантных документов по налоговому законодательству Грузии вместо частных решений.
# Проблема
* В базе 27 документов - только частные приказы и решения
* Отсутствуют базовые документы: Налоговый кодекс, Приказ 996, методологии
* Embeddings возвращают нерелевантные результаты (relevance < -2.0)
# Текущее состояние
Скрапер собирает только из раздела "ახალი დოკუმენტები" (Новые документы):
* URL pattern: [https://infohub.rs.ge/ka?species=NewDocument&page=X](https://infohub.rs.ge/ka?species=NewDocument&page=X)
* Тип документов: частные приказы и решения по конкретным делам
# Целевые разделы на infohub.rs.ge
1. **Законодательство** (საკანონმდებლო ბაზა)
    * Налоговый кодекс
    * Базовые законы
2. **Приказы МФ** (ფინანსთა სამინისტროს ბრძანებები)
    * Приказ 996 и другие нормативные приказы
3. **Методологии** (მეთოდოლოგიური განმარტებები)
    * Разъяснения налоговой службы

## 1. Исследование структуры сайта
* Определить все species types на infohub.rs.ge
* Найти URLs для каждого раздела
* Определить приоритеты по типам документов
## 2. Расширение firecrawl_scraper.py
* Добавить поддержку multiple species
* Добавить metadata: document_priority, document_category
* Добавить фильтрацию по ключевым словам (დღგ, ბრძანება 996, საგადასახადო კოდექსი)
## 3. Создание скрипта приоритетного скрапинга
* Сначала законы и кодексы (priority: high)
* Затем нормативные приказы (priority: medium)
* Последними частные решения (priority: low)
## 4. Обновление векторной базы
* Пересоздать embeddings с учетом приоритетов
* Добавить metadata filtering в RAG pipeline
* Настроить re-ranking: законы > приказы > решения
## 5. Тестирование
* Запрос "რა არის დღგ?" должен возвращать Налоговый кодекс, Приказ 996
* Проверить relevance scores > -1.0
